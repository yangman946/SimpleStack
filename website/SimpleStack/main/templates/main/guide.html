SimpleStack Guide

Last updated: June 21, 2021


Welcome to SimpleStack, this article covers: how to use SimpleStack, how it works and common issues you may encounter.

So what is SimpleStack? SimpleStack is a Search Engine dedicated to solving your programming issues. It searches the web - using Stack Exchange - for suitable solutions.


Summary


SimpleStack: searching simplified.

Question: Have you ever felt stuck? Frustrated by single line of code? No matter how hard you search you just can't seem to find the right solution?

Intoducing SimpleStack, a search engine that cares. SimpleStack will cut through all the redundancies and give you only what matters.

SimpleStack compares your search query to a range of candidates and picks out the best according to a range of criteria with the aid of AI.

To put it simply, SimpleStack makes your life easier by simplifying the debugging process.


Searching


Question: How does search work?


TIP: Once you submit a search, SimpleStack explicitly shows the Identified tags and keywords


SimpleStack will return results once you have entered a relevant query into the search bar at home. The SimpleStack algorithm will automatically break down your search query into tags and use machine learning to gather relevant results from the Stack Exchange API.


NOTE: To override this function see Key words and tagging.


The SimpleStack algorithm takes many factors into account such as but not limited to: upvotes, views and relevancy. This is done by comparing all possible results to your search query and picking out the best.

However, this process does not always guarentee a reliable or useful result. There are many factors that determines it's success, mostly dependent on how you write your query.

Generally good search results contain many key words, e.g. Print text C# is a better search than How do I show text on windows using the C# language.


TIP: SimpleStack automatically omits redundant words


The applications of efficient searching is integral in software development. The SimpleStack algorithm can judge the efficiency of the search based on your query. To ensure your results are relevant, you should be as concise as possible, replacing redundant phrases or words for important key words instead since NOT ALL redundancies are omitted.

Once you have submitted a search, various details such as the language and type of query is identified by the AI. SimpleStack will obtain results for your query and in doing so, it will compare each result to your specified query. Hence it is imperative that your search contains the necessary key words.



If your search returns no results, this means that the search query is either too specific or too vague. To fix this, rectify your search accordingly, E.g. the query NameError: name is not defined is a good search query, however multiple strings were omitted to suit the algorithm, here's the original query: Traceback (most recent call last): File "<pyshell#0>", line 1, in <module> asdf NameError: name 'asdf' is not defined.


Key words and tagging


In searching, the SimpleStack algorithm looks for key words in your search query. These words are separated and stored to be compared alongside potential results.

To increase the chance of a successful search, you should try include as many keywords as possible, the algorithm will omit as many redundancies as possible. This can include redundant words such as: to, can, in, is, how, etc.. Although the algorithm ignores these words as key words, the query input for the API will not.

Furthermore, when posting your error messages, try to simplify/generalise them, the engine doesn't need to know on what line you encountered the error or the name of your project unless absolutely necessary e.g. Traceback (most recent call last): File "<pyshell#0>", line 1, in <module> asdf NameError: name 'asdf' is not defined.

You can also take advantage of the key word functions to customize your search:



To narrow down results, utilise [tags] in your query. To use tags, simply include your tags in square brackets, e.g., [C#][.net] Print text. Tags can be used to specify the programming language or any essential information regarding your query. NOTE: this will override the automated process of tagging!

FAQ


Q: Why does a warning show when I see my search results?

A: The warning 'the following results may be unreliable as detected by the confidence algorithm...' is evoked when the top search result is not considered a good result. This can be due to a bad search query or if the topic is too broad.


Q: No results show, what do I do?

A: There are two reasons for why no results are returned on search.

  1. You search query was either too specific or too vague, so no results can be found
  2. The quota Count has been exceeded (this rarely occurs)

Q: Tags don't work, what do I do?

A: Tags are case sensitive, make sure you type them exactly the way they are shown Here.
